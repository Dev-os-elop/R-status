# Changelog

## 0.4.7 - 2026-07-10

- 자동 재실행을 셸 백그라운드 작업 대신 macOS launchd 작업으로 변경
- 앱과 설치기 프로세스에서 완전히 독립된 재시작 스크립트 추가
- 업데이트 완료 후 기존 앱이 오래 남거나 새 앱이 열리지 않던 문제 수정

## 0.4.6 - 2026-07-10

- 앱과 Addin 설치를 모두 마친 뒤 기존 프로세스를 종료하도록 순서 변경
- 설치기와 분리된 백그라운드 재시작 작업으로 새 앱 실행
- 기존 앱 종료 시 설치기까지 중단되어 파일 교체 또는 재실행이 완료되지 않던 문제 수정

## 0.4.5 - 2026-07-10

- 자동 업데이트 실행 시 현재 앱 PID를 설치기에 직접 전달
- 기존 프로세스에 종료 신호를 보낸 뒤 실제 종료 여부를 확인하고 필요 시 강제 종료
- 구버전 프로세스가 남아 새 버전 설치 후에도 이전 화면이 계속 표시되던 문제 수정

## 0.4.4 - 2026-07-10

- 진행 바의 완료·미완료 칸을 동일한 문자로 통일하고 색상만 변경
- 진행률이 바뀔 때 서로 다른 블록 문자 폭 때문에 메뉴가 좌우로 움직이던 문제 수정
- 진행 행 전체를 고정폭 글꼴로 표시해 공백과 숫자를 포함한 픽셀 폭 고정

## 0.4.3 - 2026-07-10

- 업데이트 확인을 캐시될 수 있는 main 브랜치 파일 대신 GitHub 최신 릴리스 API로 변경
- 로컬 URL 캐시를 무시하고 최신 릴리스를 강제로 다시 조회
- 릴리스 태그를 기준으로 설치 버전과 비교하도록 개선

## 0.4.2 - 2026-07-10

- Progress bar를 14칸에서 8칸으로 축소
- 현재/전체 값의 예약 폭을 total 자릿수에 맞게 축소
- 전체 고정폭 글꼴 대신 숫자만 고정폭인 시스템 글꼴 적용
- 진행 중 메뉴 폭 안정성은 유지하면서 전체 면적 축소

## 0.4.1 - 2026-07-10

- 진행률 퍼센트와 현재/전체 값을 고정 폭으로 정렬
- Progress와 Remaining 항목에 monospaced font 적용
- 남은 시간을 항상 고정 길이 `HH:MM:SS`로 표시
- 진행 메시지 길이 변화로 메뉴 폭이 흔들리지 않도록 별도 메시지 제거

## 0.4.0 - 2026-07-10

- `progress::progress_bar`와 `progressr` 진행 이벤트 자동 연동
- R Resource Usage 아래에 진행 바, 퍼센트, 현재/전체, 남은 시간 표시
- 진행 패키지를 사용하지 않을 때 관련 메뉴 항목 자동 숨김
- 진행 이벤트 전송을 약 0.25초 간격으로 제한하고 기존 progressr handler 복원

## 0.3.9 - 2026-07-10

- Check for Updates에서 새 버전을 직접 다운로드하고 설치하도록 변경
- 태그 ZIP의 설치기를 백그라운드로 실행해 앱과 Addin을 함께 업데이트
- 설치 성공 시 RStudio Status 자동 재실행
- 자동 설치 준비 또는 실행 실패 시 GitHub 수동 설치 경로 제공

## 0.3.8 - 2026-07-10

- 실행 중인 R 세션 PID를 앱에 전달하고 0.5초 간격으로 생존 상태 확인
- RStudio 앱 전체가 종료되면 메뉴 상태를 자동으로 Interrupted로 전환
- RStudio가 상태 전송 전에 종료되더라도 실행 시간 카운터 자동 정지

## 0.3.7 - 2026-07-10

- 완료·실패·중단 상태 수신 시 R 리소스 정보를 즉시 다시 샘플링
- 진행 중인 샘플이 있으면 끝나는 즉시 한 번 더 갱신하여 오래된 worker 수 제거
- 병렬 예제를 짧은 작업 batch로 나눠 RStudio Stop에 빠르게 반응하도록 개선

## 0.3.6 - 2026-07-10

- 설치 시 GitHub Release의 사전 빌드 Apple Silicon 실행 파일 사용
- 일반 사용자의 Xcode 및 Command Line Tools 설치 요구사항 제거
- 다운로드한 실행 파일의 SHA-256 무결성 검증
- 소스 빌드용 toolchain 검사를 개발 스크립트로 분리

## 0.3.5 - 2026-07-10
